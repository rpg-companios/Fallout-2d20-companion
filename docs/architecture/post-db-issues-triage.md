## Детали и предлагаемые фиксы

### #6 Супермутант (Сила) — простой, изолированный
Данные трейта: `modifiers.attributes.STR = { baseBonus:2, min:6, max:12 }`.
Код `getAttributeLimits` смотрит `modifiers.minLimits[STR]` / `maxLimits[STR]` — их нет → 4/10.
**Фикс:** в `getAttributeLimits` дополнительно читать `modifiers.attributes[attr]?.min/max`
(приоритет: minLimits/maxLimits → attributes.{x}.min/max → BASE). Проверить и применение
`baseBonus` при выборе трейта (стартовое значение Силы должно стать 6).
Риск: низкий. Только домен характера.

### #5 Сохранёнки теряют данные — важный, но осторожный
`[] ?? snapshot.attributes` сохраняет пустой массив. 
**Фикс:** менять на «брать store-данные только если они НЕ пустые»:
`legacyData.attributes?.length ? legacyData.attributes : snapshot.attributes` (и так же skills,
equipment, equippedWeapons). Это чинит затирание.
Про «картинку»: проверить отдельно — где хранится изображение origin и попадает ли в snapshot
(возможно, image вообще не в снапшоте). Нужно уточнение/проверка.
Риск: средний (трогает save/load). Обязательно тест на round-trip save→load.

### #3 Двоение предметов — нужно воспроизвести точечно
`handleAddItem` стекает по `stackKey`. Двоение = либо `getStackKey` даёт разные ключи
одинаковым предметам, либо предмет показывается в двух списках (инвентарь + экипировка).
**Нужно:** какой именно предмет двоился (оружие/броня/хлам?), и двоится в инвентаре или
между инвентарём и «надето». Без этого — гадание.

### #2 Гипнотрон / встроенное оружие головы — войдёт в рефактор рук
Тот же корень, что протектрон манипулятор+лазер: при пересборке списка атак встроенное
оружие (головы/руки) теряется. Чистая модель `weaponIds` (Шаги 5–8) решает это системно.

## Рекомендованный порядок (атомарно, по одному)
1. **#6** супермутант — маленький изолированный фикс + тест.
2. **#5** сохранёнки — фикс затирания + round-trip тест (важно для данных).
3. **#3** двоение — сначала воспроизвести (нужны детали), потом фикс.
4. **#2** + рефактор рук (Шаги 5–8) — системно, отдельным крупным патчем.

Каждый — отдельный патч, отдельный коммит, прогон тестов.
