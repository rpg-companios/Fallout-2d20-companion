# Управление конечностями в инвентаре

## Обзор

Данный документ описывает отложенную задачу по управлению конечностями роботов в инвентаре. В рамках спецификации `robot-equipment-refactoring` было принято решение **не реализовывать** эту функциональность в текущей итерации: конечности скрыты из инвентаря и управляются исключительно через экран `WeaponsAndArmorScreen` (кнопка «Модернизировать конечность»).

Настоящий документ фиксирует требования и подходы к реализации для будущей итерации.

> Связанные требования: **14.1–14.3** из `robot-equipment-refactoring/requirements.md`

---

## Текущее поведение (после `robot-equipment-refactoring`)

- Предметы с `itemType` в `['robotArm', 'robotHead', 'robotBody', 'robotLeg']` **фильтруются** из списка инвентаря (`InventoryScreen`) и не отображаются игроку.
- Замена конечности через `LimbUpgradeModal` перемещает старую конечность в `equipment.items`, но она остаётся невидимой.
- Кнопка «Снять конечность» в `WeaponsAndArmorScreen` отсутствует.

---

## Задача 1: Отображение снятых конечностей в инвентаре

### Описание

После замены конечности через `LimbUpgradeModal` старая конечность попадает в `equipment.items`. Необходимо сделать её видимой в `InventoryScreen` с соответствующей карточкой предмета.

### Условие отображения

Конечность отображается в инвентаре **только если** она была явно снята/заменена игроком, то есть не является конечностью по умолчанию из стартового комплекта, которая никогда не надевалась вручную.

Технически: конечность отображается, если она присутствует в `equipment.items` (туда она попадает только при замене через `applyLimbReplacement`).

### Изменения

**`InventoryScreen`:**
- Убрать фильтрацию `robotArm | robotHead | robotBody | robotLeg` из списка инвентаря (или сделать её условной).
- Отображать конечности в отдельной секции «Конечности» или в общем списке с иконкой типа.

**Карточка конечности в инвентаре:**
- Название конечности.
- Тип слота (голова / корпус / рука / нога).
- Краткие характеристики (встроенное оружие, `canHoldWeapons`, совместимые шасси).
- Кнопка «Экипировать» (см. Задача 2).

### Требования к данным

Конечности уже содержат необходимые поля (`itemType`, `compatibleBodyPlans`, `defaultForBodyPlan`, `builtinWeaponId`). Дополнительных изменений в JSON не требуется.

## Задача 2: Экипировка конечностей из инвентаря

### Описание

Игрок должен иметь возможность надеть конечность из инвентаря в подходящий слот робота.

### Поток взаимодействия

1. Игрок нажимает «Экипировать» на конечности в инвентаре.
2. Открывается диалог выбора слота — показываются только совместимые слоты (по `itemType` и `compatibleBodyPlans`).
3. Игрок выбирает слот.
4. Вызывается `applyLimbReplacement(slots, slotKey, newLimb)`:
   - Старая конечность из слота перемещается в `equipment.items`.
   - Новая конечность устанавливается в слот.
   - Пересобирается `equippedWeapons`.
5. Конечность удаляется из `equipment.items`.

### Проверки совместимости

Перед отображением слотов вызывается `canReplaceLimb(slotKey, newLimb, character)`:
- Проверяется соответствие `itemType` типу слота (например, `robotArm` → `leftArm` / `rightArm`).
- Проверяется `compatibleBodyPlans` или `defaultForBodyPlan` относительно текущего шасси робота.
- Несовместимые слоты отображаются как неактивные с пояснением причины.

### Изменения

**`InventoryScreen`:**
- Добавить обработчик «Экипировать» для конечностей.
- Реализовать диалог выбора слота (новый компонент `LimbSlotSelectModal` или встроенный `Alert`).

**`domain/robotEquip.js`:**
- Функции `canReplaceLimb` и `applyLimbReplacement` уже реализованы — использовать без изменений.

**`CharacterContext`:**
- Обновить `setEquippedRobotSlots` и `setEquipment` после экипировки.
- Обновить `setEquippedWeapons` через `getBuiltinWeaponsFromSlots`.

## Задача 3: Кнопка «Снять конечность» в `WeaponsAndArmorScreen`

### Описание

В компоненте `RobotSlot` добавить кнопку «Снять конечность», позволяющую снять текущую конечность из слота и переместить её в инвентарь.

### Поведение

- При нажатии «Снять конечность»:
  1. Конечность из `slots[slotKey].limb` перемещается в `equipment.items`.
  2. Слот `limb` устанавливается в `null`.
  3. Если у конечности было встроенное оружие (`builtinWeaponId`) или она была манипулятором — соответствующее оружие удаляется из `equippedWeapons`.
  4. Если в слоте было `heldWeapon` — оружие возвращается в инвентарь (слот `heldWeapon` → `null`).
  5. Пересобирается `equippedWeapons` через `getBuiltinWeaponsFromSlots`.

### Ограничения

- Кнопка **недоступна** (disabled), если конечность является конечностью по умолчанию из стартового комплекта и в инвентаре нет замены. *(Опционально — на усмотрение геймдизайнера.)*
- Кнопка **недоступна**, если слот `limb` уже `null`.

### Изменения

**`components/screens/CharacterScreen/modals/LimbUpgradeModal.js`** (или `RobotSlot`):
- Добавить кнопку «Снять конечность» рядом с «Модернизировать конечность».
- Обработчик вызывает новую функцию `removeLimb(slots, slotKey)` из `domain/robotEquip.js`.

**`domain/robotEquip.js`:**
- Добавить функцию `removeLimb(slots, slotKey)`:
  ```js
  // Снимает конечность из слота, возвращает обновлённые слоты,
  // пересобранный массив оружия и предмет для инвентаря
  removeLimb(slots, slotKey) → {
    slots: RobotSlotsObject,
    weapons: WeaponObject[],
    removedLimb: LimbObject | null,
    removedWeapon: WeaponObject | null  // heldWeapon если был
  }
  ```

**`CharacterContext`:**
- Обновить `equippedRobotSlots`, `equippedWeapons`, `equipment.items` после снятия.

## Взаимосвязь задач

```
Задача 3 (Снять конечность)
    ↓ конечность попадает в equipment.items
Задача 1 (Отображение в инвентаре)
    ↓ игрок видит конечность и нажимает «Экипировать»
Задача 2 (Экипировка из инвентаря)
    ↓ конечность возвращается в слот
```

Задачи реализуются в указанном порядке: без Задачи 3 нет смысла в Задачах 1 и 2.

---

## Примечания по реализации

- Все доменные функции (`canReplaceLimb`, `applyLimbReplacement`, `getBuiltinWeaponsFromSlots`) уже реализованы в `domain/robotEquip.js` — новая функция `removeLimb` является их логическим дополнением.
- Фильтрация конечностей в `InventoryScreen` реализована в задаче 12 спецификации `robot-equipment-refactoring` — при реализации данного документа её нужно сделать условной (показывать конечности, попавшие в инвентарь через `removeLimb` / `applyLimbReplacement`).
- Стартовые конечности из комплекта снаряжения, которые никогда не снимались, **не должны** появляться в инвентаре — это сохраняет текущее поведение (требование 14.3).
