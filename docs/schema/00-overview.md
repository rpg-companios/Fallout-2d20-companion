# v1.5 — Контракт данных (источник правды для рефакторинга)

Это **контракт**: поля здесь — одновременно ИСТОЧНИК изменений (что движок читает)
и ЦЕЛЬ изменений (к чему приводим данные). Любая правка кода/данных сверяется с этим
документом. Перед кодом — обновляем контракт.

## Принципы
- Данные — в JSON (`data/`), логика — в `domain/`, состояние — в Zustand, экраны — отдельно, i18n — отдельно.
- Каждый файл схемы описывает ОДНУ сущность: origins, traits, attributes, skills, derived, kits…
- Архивируем старое **поэтапно**: заменили блок логики → старое в `old_logic/`. Не ломаем рабочее на полпути.
- Будущее (не сейчас): единая система модификаторов (см. ../architecture/modifiers-vision.md).

## Порядок работ (согласован с владельцем)
1. **Origins** — типы персонажей + базовые правила по типу.  ← начинаем здесь
2. **Traits** — какие модификаторы, на что влияют.
3. **Attributes & Skills** — что на них влияет.
4. **Derived** — производные характеристики.
5. **Equipment kits** — комплекты снаряжения.
